9. Aplikasi SIG ada 4 (empat), salah satunya adalah aplikasi pemodelan (modelling). Sebutkan salah satu aplikasi pemodelan dan jelaskan secara singkat bagaimana proses pemodelannya !

Overlay  adalah teknik yang menggabungkan beberapa data geospasial atau peta yang mempunyai referensi sama untuk menghasilkan data atau peta baru. Pendekatan yang digunakan dalam teknik overlay antara lain matriks dua dimensional, kuantitatif binary, kuantitatif berjenjang, dan kuantitatif berjenjang tertimbang. Syarat penggunaan pendekatan matriks dua dimensional ialah data memiliki isi dan wilayah kajian yang sama, tetapi memiliki tahun yang berbeda.

Overlay dilakukan dengan proses :

1) Memasukaan data spasial yang akan di overlay data tersebut bisa dalam bentuk peta analog, data penginderaan jauh, data hasil pengukuran lapangan dan data GPS.

2) Menyesuaikan sistem koordinat yang digunakan harus sama pada setiap data yang di overlay

3) Menyusun letak data spasial agar semua data yang di overlay terlihat dan tidak tertutupi dengan data lainnya

4) Menyusun data polygon berada pada bagian paling bawah agar data yang berupa point, polyline dan data data yang disimbolkan kecil agar terlihat pada output petanya nanti

12. notasi grafis dalam pemodelan data konseptual yg digunakan untuk memodelkan struktur data dan hubungan antardata disebut


Jawaban:

Notasi Grafis dalam pemodelan data konseptual sebagai suatu karakteristik atau ciri yang membedakan antara entitas satu dengan entitas yang lainnya adalah Entity Relationship Diagram/Diagram Hubungan Entitas.

Pembahasan

Permodelan data terdiri dari model hubungan entitas dan model relasional. Diagram hubungan entitas ditemukan oleh Peter Chen dalam buku Entity Relational Model-Toward a Unified of Data.

Entitas adalah suatu objek yang dapat didefinisikan dalam lingkungan pemakai dalam konteks sistem yang akan dibuat.

Adapun Kegunaan Diagram Hubungan Entitas yaitu :

Untuk mengkonstruksikan model data konseptual

Untuk memodelkan struktur data dan hubungan antar data Untuk mengimplementasikan basis data secara logika maupun secara fisik dengan Database Management System (DBMS).

Untuk membantu dalam menjawab persoalan tentang data yang diperlukan dan bagaimana data tersebut saling berhubungan.

Symbol-symbol yang ada dalam Diagram Hubungan Entitas, yaitu :

Persegi panjang, menyatakan himpunan entitas.

Oval, menyatakan atribut (atribut key digaris bawahi).

Belah ketupat, menyatakan himpunan relasi.

Garis, menyatakan penghubung antara himpunan relasi dengan himpunan entitas dan himpunan entitas dengan atributnya

USER INTERFACES

 

Jawaban  

 

Pembahasan  

User interface adalah media komunikasi antara user dengan sistem. User dan sistem akan saling bertukar informasi di dalamnya.

Tujuan user interface antara lain:

membuat interface menjadi seefisien mungkinmemperbaiki produktivitas para usermemudahkan pekerjaan user

Manfaat user interface:

membuat tampilan yang bersahabat dan efisien agar memudahkan para user menyelesaikan tugasnya.

Ada beberapa kualitas yang diperhatikan dalam user interface:

Kejelasan dari user interface tanpa menimbulkan ambiguitas.

Keringanan user interface

Familiarity, atau tampilan yang bersahabat

Dapat memberikan umpan balik yang tepat kepada pengguna/user

Konsisten pada tiap aplikasi

Terlihat bagus, mempunyai estetika dan nyaman dilihat

Adanya shortcut-shortcut yang membantu mempercepat pekerjaan

Adanya sarana untuk memperbaiki kesalahan.


Tipe-tipe User Interface:

Command-Line Interface: input yang diberikan berupa perintah dalam bentuk teks dengan keyboard komputer.

Direct Manipulation Interface: input yang diberikan berupa gambar-gambar.

Touchscreen: input yang diterima berupa sentuhan.

Gestures Interface: input yang diterima berupa gerakan tangan.

Search Interface: memunculkan kotak pencarian dan mempresentasikan hasil pencariannya.

Natural Language Interface: Penggunaan bentuk pertanyaan dan jawaban pada kotak pencarian.

Voice User Interface: input dan output berupa perintah suara.

38. Jelaskan perbedaan pemodelan bangkitan dan tarikan dengan pemodelan sebaran pergerakan!


Pemodelan bangkitan dan tarikan bertujuan untuk memperkirakan jumlah pergerakan yang akan dilakukan dari suatu zona asal dan jumlah pergerakan yang tertarik ke dalam zona tujuan.

Sedangkan pemodelan sebaran yaitu bertujuan untuk menjelaskan proses antara interaksi antar penggunaan lahan, penggunaan transportasi, dan arus lalu lintas.

Pembahasan

Dalam perencanaan transportasi yaitu terdapat pemodelan bangkitan dan tarikan pergerakan (trip generation). Bangkitan dan tarikan pergerakan (trip generation) yang merupakan tahapan pemodelan yang bertujuan untuk memperkirakan jumlah pergerakan yang akan dilakukan dari suatu zona asal dan jumlah pergerakan yang tertarik ke zona tujuan. Ada beragam pergerakan yang terjadi misalnya pergerakan bisnis, berbelanja, pendidikan, wisata dan lain-lain.

Sedangkan pemodelan sebaran (trip distribution) yaitu merupakan pemodelan yang bertujuan menjelaskan interaksi antar penggunaan lahan, penggunaan transportasi, dan arus lalu lintas atau utnuk menjelaskan bagaimana terjadinya lalu lintas tersebut.

Jadi, perbedaan pemodelan bangkitan dan tarikan dengan pemodelan sebaran yaitu memiliki perbedaan pada aspek yang dijelaskan.
