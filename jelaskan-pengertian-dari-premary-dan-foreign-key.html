1. Jelaskan perbedaan antara primary key dan foreign key!


Prymary Key : suatu atribut atau satu set minimal atribut yang tidak hanya mendefinisikan secara unik suatu kejadian spesifik tetapi juga dapat mewakili setiap kejadian dari suatu kejadian.

Foreign Key  : satu set atribut atau set atribut sebagai key penghubung kedua tabel dan melengkapi satu relationship (hubungan) terhadap primary key yang menunjukan keinduknyaPrimery key-Kunci utama yg rahasia
Foreign key- kunci tamu untuk melihat informasi di dalam data base

2. jelaskan syarat foreign key


FK Menunjuk ke Primary Key dari table lain Pertama, foreign key itu merupakan sebuah primary key dari suatu tabel yang berada pada tabel lain yang berfungsi untuk menunjukkan adanya hubungan relasi antar tabel tersebut.

Jika dilihat dari pengertian di atas, maka dapat disimpulkan bahwa syarat dari foreign key diantaranya:
- Merupakan primary key yang berasal dari tabel lain.
- Berfungsi sebagai kunci (key) untuk membuat relasi antar tabel.

3. Jelaskan yang dimaksud dengan Composite Key, Superkey , Candidate Key , Primary Key, Foreign!


Jawaban:

1. composite key

composite key merupakan sebuah kunci yang di dalamnya terdiri dari dua atau lebih atrbut. Atribut-atribut tersebut akan secara unik melakukan identifikasi terhadap sebuah kejadian entitas.

2. Primary Key

Primary Key juga dinamakan sebagai kunci utama di dalam sebuah tabel pada database. Keberadaan kunci ini memang sangat diperlukan karena berperan sebagai identitas yang nantinya akan merepresentasikan keseluruhan isi tabel. Dengan adanya kunci utama ini, maka proses pengolahan data menjadi lebih mudah. Primary key juga berperan penting di dalam menghubungkan antara dua tabel atau lebih.

3. foreign key

Foreign key atau yang juga dinamakan kunci asing atau kunci tamu merupakan sebuah kunci utama yang ada di tabel lainnya. Fungsinya adalah untuk merelasikan antar tabel.

4. Alternate key

Sebuah kunci yang tidak dipilih. Misalnya, ada dua kandidat yang akan menjadi kunci utama. Salah satunya pasti akan ditunjuk sebagai primary key. Sementara kunci yang tidak dipilih tersebut disebut sebagai alternate key

5. Candidate Key

Kunci yang berperan untuk mengidentifikasi adanya kejadian spesial pada tabel. Syarat sebuah kunci dinamakan candidate key adalah unik identifier, serta non-duplikat. Maksudnya, tidak ada kunci yang memiliki ciri khas yang sama dengan candidate key.

6. Super Key

Istilah lainnya yang juga tidak kalah penting adalah super key. Super key berperan dalam memberikan perbedaan untuk setiap baris data secara unik di dalam tabel.


4. Apa yang disebut foreign key


Dalam konteks basis data relasional, foreign key adalah bidang dalam satu tabel yang secara unik mengidentifikasi deretan tabel lain atau tabel yang samaadalah kolom atau field pada suatu tabel yg berfungsi sebagai kunci tamu dari tabel lain

5. tolong jelaskan perbedaan superkey, candidate key dan primary key :)


1. Primary Key

Primary Key merupakan sebuah aturan dimana fungsinya adalah untuk membedakan anatara baris satu dengan baris lainnya yang ada pada tabel dan bersifat unik. Syarat :

*Data tidak boleh sama atau ganda (unik).

*Data tidak boleh bernilai null

2. Candidate Key

andidate key merupakan suatu atribut ataupun super key yang mengidentifikasi secara unik untuk kejadian spesifik dari entitas.

3. Super Key

suatu atribut untuk melengkapi hubungan yang menunjukan ke induknya, itu artinya field pada tabel merupakan kunci tamu dari tabel lain. Dan biasanya penggunaan foreign key akan sangat dibutuhkan ketikan kita menemukan banyak tabel dan ingin menghubungkan satu tabel dengan tabel lainnya.


6. Jelaskan dan sebutkan fungsi dari tombol-tombol keyboard: a.numeric key b.alphabet key c.function key d.special function key


Jelaskan dan sebutkan fungsi dari tombol-tombol keyboard:a.numeric keyb.alphabet keyc.function keyd.special function keyJawaban Pendahuluan

Soal diatas adalah soal mengenai beberapa tombol pada keyboard, seperti kita ketahui bahwa keyboard merupakan salah satu perangkat input pada komputer. Beberapa tombol pada keyboard memiliki fungsi khusus.

Pembahasan

Beberapa tombol khusus pada keyboard yang akan kita bahas adalah sebagai berikut

a. Numeric key, numeric key pada keyboard berfungsi untuk memberikan inputan berupa angka dan operator matematika seperti penjumlahan dan pengurangan dan masih banyak lainnya.

b. Alphabet key, berfungsi untuk memberikan inputan berupa huruf dan tanda baca. Inputan angka juga termasuk ke dalam alphabet key.

c. Function key, berfungsi untuk mengaktifkan menu tertentu pada suatu aplikasi ketika tombol tersebut ditekan. Function keyboard pada keyboard adalah tombol F1 hingga F12 yang biasanya terdapat pada bagian atas keyboard.

d. Special function key, tombol ini bukan merupakan tombol untuk memberikan inputan. Tombol special function key hampir sama dengan function key, namun tombol ini berfungsi ketika ditekan bersamaan dengan tombol lain. Tombol special funtion key antara lain adalah CTRL, ATL. Misal untuk memunculkan task manager adalah dengan menekan CTRL+ALT+DEL.

Kesimpulan

Demikian pembahasan mengenai beberapa tombol pada keyboard, dengan memahami tombol-tombol pada keyboard maka dapat meningkatkan kecepatan kita dalam menyelesaikan suatu pekerjaan atau tugas. 27. Jelaskan secara singkat dan berikan contoh pengertian entitas,atribut dan key(super key,candidat key,primary key)?


Candidate Key (Kunci Kandidat)

Database dalam relational database dapat disederhanakan sebagai sekumpulan tabel yang saling terhubung. Setiap baris dari dalam tabel setidaknya harus memiliki sebuah kolom yang unik. Unik disini maksudnya tidak boleh sama. Contohnya : tabel data_mahasiswa, kolom NIM (Nomor Induk Mahasiswa) akan menjadi kandidat yang bagus, karena tidak mungkin ada 2 mahasiswa yang memiliki NIM yang sama. NIM disini disebut juga dengan Candidate Key (Kunci Kandidat).

Candidate Key adalah satu atau beberapa kolom dalam tabel yang bisa mengidentifikasi tiap baris dari tabel tersebut.

Beberapa karakteristik Candidate key : unik (tidak boleh berulang), tidak boleh memiliki nilai null (kosong), nilai dari candidate key akan sangat jarang berubah.

Primary Key (Kunci Primer)

Kunci primer adalah Dalam sebuah tabel, akan terdapat beberapa candidate key, namun hanya ada 1 Primary key (kunci utama). Primary key adalah salah satu candidate key yang kita nobatkan sebagai kolom unik untuk identifikasi baris dalam tabel.

Nilai field yang menjadi primary key harus:

– Unik atau tidak boleh ganda

– Tidak boleh Null (kosong, tidak diketahui, tidak dapat ditentukan)

– Key tersebut lebih natural untuk dijadikan acuan

Contoh Primary Key :

File pegawai berisi attribute :

no induk, No ktp, Nama, Tempat lahir, Tanggal lahir, Alamat, Kota

No induk dan no ktp adalah kunci calon (Candidate Key) dan untuk kunci utama (primary key) adalah salah satu yang dipilih dari kunci calon. Misalnya No. induk di jadikan primary key, maka primary key nya adalah no induk.

Foreign Key (Kunci Tamu)

Foreign Key adalah satu set atribut atau set atribut sebagai key penghubung kedua tabel dan melengkapi satu relationship (hubungan) terhadap primary key yang menunjukan keinduknya. Jika sebuah primary key terhubungan ke table/entity lain, maka keberadaan primary key pada entity tersebut di sebut sebagai foreign key.

Super Key adalah sebuah atau sekumpulan atribut yang secara unik mengidentifikasi sebuah tupel dalam tabel relasi. Berdasarkan contoh tabel MHS sebelumnya, super key yang mungkin adalah (NIM), (NIM, Nama), (NIM, Alamat), (Nama, Alamat), dan (NIM, Nama, Alamat).

37. ada yang disebut primary key(kunci utama),kandidat key kemudian jelaskan apa bedanya​


Jawaban:

Candidat Key adalah adalah suatu atribut atau satu set minimal atribut yang hanya mengidentifikasikan secara unik untuk suatu kejadian spesifik dari entitas.

Primary key merupakan candidat key yang telah dipilih untuk mengidentifikasikan secara unik untuk suatu kejadian spesifik dari entitas.

Perbedaannya:

Candidat Key bisa terdapat lebih dari satu dalam satu tabel, sedangkan primary key hanya satu yaitu candidat key yang telah dipilih. Jadi, primary key sudah pasti candidat key, sedangkan candidat key belum tentu primary key tapi salah satunya pasti ada yang menjadi primary key.
